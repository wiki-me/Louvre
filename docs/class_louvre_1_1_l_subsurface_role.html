Detailed Description

Subsurface role for surfaces.

The LSubsurfaceRole class defines a role for surfaces that allows them to be positioned relative to their parent surfaces. They are always children of other surfaces and get their position based on their parent's position plus an offset defined by the localPos() function.

The Subsurface role is part of the Wayland protocol.

Modes

Subsurfaces can work in two modes:

Synchronous Mode

In this mode, changes to the subsurface, such as its position, are applied exclusively when its parent performs a commit.
This mode is used by clients to synchronize the animation or movement of multiple subsurfaces.
The library keeps track of changes and applies them when the parent performs a commit.

Asynchronous Mode

In this mode, subsurface changes are applied independently of the commits of its parent.

virtual const LPoint& rolePos ( ) const

Position of the subsurface according to the role.

The default implementation of rolePos() positions the subsurface relative to its parent by adding the offset given by localPos().
Reimplement this virtual method if you want to define your own logic for positioning the subsurface.

Default implementation

const LPoint &LSubsurfaceRole::rolePos() const
{
if (surface()->parent())
m_rolePos = localPos() + surface()->parent()->rolePos();
return m_rolePos;
}
